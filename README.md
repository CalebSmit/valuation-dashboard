# AI Stock Valuation Dashboard

An AI-powered equity valuation dashboard that reads your existing financial data and produces institutional-quality DCF, DDM, and comparable company analysis with cited assumptions.

## Start Here

- Fastest user path: `QUICKSTART.md`
- Deployment and environment setup: `DEPLOYMENT.md`
- Combined release verification: `VERIFY_DASHBOARD.bat`

## Prerequisites

- Python 3.11+ with pip
- Node.js 18+ with npm
- An Anthropic API key (for AI-powered assumption generation)
- Financial data generated by the main pipeline (`py main.py`)

## Quick Start

### 1. Generate Financial Data (if not already done)

```bash
# From the project root (My-Finance-Project/)
py main.py
```

This creates `raw_data.xlsx` with 48+ sheets of financial data.

### 2. Install Dashboard Dependencies (first time only)

```bash
# Backend
cd valuation-dashboard/backend
pip install -r requirements.txt

# Frontend
cd ../frontend
npm install
```

Optional: copy `.env.example` values into your local shell or deployment environment if you want backend-owned provider keys or non-default ports/origins.

### 3. Launch the Dashboard

**Option A: Use the batch file (Windows)**
```bash
cd valuation-dashboard
START_DASHBOARD.bat
```

The launcher now waits for a healthy backend before opening the frontend.

**Option B: Start manually (two terminals)**
```bash
# Terminal 1: Backend
cd valuation-dashboard/backend
py -m uvicorn main:app --port 8000

# Terminal 2: Frontend
cd valuation-dashboard/frontend
npm run dev
```

### 4. Open the Dashboard

Navigate to **http://localhost:5173** in your browser.

- On first launch, you'll be prompted to enter your Anthropic API key
- If the backend already has a provider key configured, entering a browser key is optional
- Enter a stock ticker and click **ANALYZE**
- The AI agent generates assumptions, then the dashboard computes DCF, DDM, and comps valuations

## Features

- **DCF Valuation** with WACC build-up, 5-year FCF projections, and sensitivity analysis
- **Dividend Discount Model** with applicability check and single/two-stage pricing
- **Comparable Company Analysis** with peer multiples and implied prices
- **Bear/Base/Bull Scenario Analysis** with explicit assumption drivers
- **Football Field Chart** showing all valuation methods vs current price
- **Editable Assumptions** with instant recalculation (<50ms)
- **Excel Export** matching the Smit Financial Model structure
- **Run History** saved locally in your browser (IndexedDB)

## How It Works

1. The dashboard reads `raw_data.xlsx` (generated by `py main.py`)
2. The AI agent (Claude) analyzes the financial summary and sets defensible assumptions with citations
3. All valuation calculations run locally in your browser (TypeScript)
4. You can override any assumption and recalculate instantly
5. Export to Excel for presentations

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "raw_data.xlsx not found" | Run `py main.py` from the project root first |
| Backend won't start | Run `pip install -r requirements.txt` in the backend directory |
| Frontend won't start | Run `npm install` in the frontend directory |
| "API key required" | Enter your Anthropic API key in the Settings modal |
| Port 8000 in use | Kill the existing process or change the port in `config.py` |

## Deployment

### Local (default)

Both frontend and backend run on your machine. No extra configuration needed — Vite proxies `/api` to `localhost:8000` automatically.

### Split deployment (e.g. Cloudflare Pages + Fly.io)

The frontend (static React build) and backend (Python/FastAPI) are deployed separately:

| Part | Where | Notes |
|------|-------|-------|
| Frontend | Cloudflare Pages, Netlify, Vercel, etc. | Static files only — `npm run build` output |
| Backend | Fly.io, Render, Railway, a VPS, etc. | Needs Python, file system access for `raw_data.xlsx` |

**Backend setup:**
1. Deploy the `backend/` folder to any Python host
2. Set env vars: `ANTHROPIC_API_KEY`, `VALUATION_DASHBOARD_ALLOWED_ORIGINS=https://your-frontend.pages.dev`
3. Upload `raw_data.xlsx` to the server (or trigger the pipeline remotely)

**Frontend setup (Cloudflare Pages):**
1. Set build command: `cd frontend && npm install && npm run build`
2. Set output directory: `frontend/dist`
3. Add environment variable: `VITE_API_BASE_URL=https://your-backend.fly.dev`

Cloudflare Pages itself only hosts the static files. The Python backend **cannot** run on Cloudflare Workers/Pages — it requires a separate Python host.

See [DEPLOYMENT.md](DEPLOYMENT.md) for full environment variable reference.
